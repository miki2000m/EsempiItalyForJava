Riepilogo: StringBuilder (in italiano)

Cosa è
- StringBuilder è una classe mutabile per la manipolazione efficiente di sequenze di caratteri.
- A differenza di String, le operazioni su StringBuilder modificano l'oggetto esistente senza crearne di nuovi.

Principali metodi
- append(...) : aggiunge contenuto alla fine. Supporta tipi primitivi, charSequence, String, ecc.
- insert(index, ...) : inserisce contenuto alla posizione specificata.
- replace(start, end, String) : sostituisce la porzione [start, end) con la stringa data.
- delete(start, end) : rimuove la porzione [start, end).
- reverse() : inverte l'ordine dei caratteri.
- toString() : converte in una String immutabile.
- charAt(index) / setCharAt(index, char) : legge/modifica il carattere in posizione index.
- length() : numero di caratteri attuali.
- capacity() : capacità attuale dell'array interno (>= length).
- ensureCapacity(min) : garantisce che la capacità sia almeno min.

Mutabilità e thread-safety
- StringBuilder è mutabile e NON è thread-safe (non sincronizzato).
- Per casi multi-threaded dove serve sincronizzazione, usare StringBuffer (sincronizzato), oppure gestire la sincronizzazione esternamente.

Quando preferire StringBuilder
- Quando si costruiscono ripetutamente stringhe (concatenazioni in loop), StringBuilder evita la creazione continua di oggetti String temporanei.
- Per performance in contesti single-thread.

Note
- La capacity iniziale di new StringBuilder() è 16. Se si costruisce una stringa di dimensione nota, si può usare new StringBuilder(initialCapacity) per evitare riallocazioni.
- Anche StringBuffer offre API simili ma è sincronizzato.

Esempio rapido di esecuzione
- Dal root del progetto (cartella che contiene src), compilare:
  javac src/MODULO_3_API_FONDAMENTALI/String/MainStringBuilder.java
- Eseguire (impostando la classpath corretta):
  java -cp src MODULO_3_API_FONDAMENTALI.String.MainStringBuilder
