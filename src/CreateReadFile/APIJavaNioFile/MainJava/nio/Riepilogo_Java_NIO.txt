RIEPILOGO CONCETTI E COMANDI - LIBRERIA JAVA.NIO.FILE

Questo file contiene un riassunto dei concetti teorici e dei comandi principali discussi nel file guida interattivo.

--- CONCETTI TEORICI FONDAMENTALI ---

*   Filesystem: Lo "schedario" del sistema operativo per organizzare file e cartelle.
*   Directory: Una "cartella" che contiene file e altre directory.
*   Root Directory: La cartella principale del filesystem (es. `C:\` o `/`).
*   Path: L'"indirizzo" di un file o di una directory. È un oggetto che rappresenta la posizione, non il file stesso.
*   Charset (es. UTF-8): Un "dizionario" che converte i caratteri di testo in numeri che il computer può salvare. UTF-8 è lo standard universale che garantisce la leggibilità dei file su sistemi diversi.
*   Override: In programmazione a oggetti, significa fornire una propria implementazione specifica per un metodo che è già definito in una classe genitore (superclasse).

--- RIEPILOGO COMANDI PER SEZIONE ---

**Sezione 1: Path e Paths**
*   Paths.get(base, ...parts) -> Crea un oggetto Path combinando delle stringhe.
*   path.getFileName() -> Estrae il nome del file o della directory finale dal percorso.
*   path.getParent() -> Restituisce il percorso della directory genitore.
*   path.getRoot() -> Restituisce la radice del percorso (es. `/` o `C:\`).
*   path.resolve(other) -> Crea un nuovo percorso unendo il percorso attuale con un altro (naviga "in basso").
*   path.normalize() -> Semplifica un percorso, risolvendo `.` e `..`.

**Sezione 2: Operazioni di Base**
*   Files.createFile(path) -> Crea un nuovo file vuoto. Fallisce se esiste già.
*   Files.createDirectory(path) -> Crea una nuova directory. Fallisce se esiste già o il genitore non esiste.
*   Files.createDirectories(path) -> Crea una directory e tutte le cartelle genitore necessarie. Non fallisce se esiste già.
*   Files.exists(path) -> Controlla se un file o una directory esiste.
*   Files.isDirectory(path) -> Controlla se il percorso è una directory.
*   Files.delete(path) -> Cancella un file o una directory (che deve essere vuota). Lancia un'eccezione se non esiste.
*   Files.deleteIfExists(path) -> Cancella un file/dir, ma non lancia un'eccezione se non esiste.

**Sezione 3: Scrittura e Lettura**
*   Files.write(path, righe, charset) -> Scrive una lista di stringhe su un file (sovrascrive).
*   Files.readAllLines(path, charset) -> Legge tutte le righe di un file in una List<String>.
*   Files.newBufferedWriter(path, ...) -> Ottiene uno stream per scrivere testo in modo efficiente (con buffer).
*   writer.newLine() -> Scrive un carattere di a-capo.
*   Files.newBufferedReader(path, ...) -> Ottiene uno stream per leggere testo in modo efficiente (con buffer).
*   reader.readLine() -> Legge una singola riga di testo dallo stream.

**Sezione 4: Attributi dei File (Metadati)**
*   Files.size(path) -> Restituisce la dimensione del file in byte.
*   Files.getLastModifiedTime(path) -> Restituisce la data e l'ora dell'ultima modifica.
*   Files.readAttributes(path, Type.class) -> Legge un blocco di attributi in modo efficiente in un solo colpo.

**Sezione 5: Copia e Spostamento**
*   Files.copy(source, target, options) -> Copia un file. `StandardCopyOption.REPLACE_EXISTING` sovrascrive se la destinazione esiste.
*   Files.move(source, target, options) -> Sposta (o rinomina) un file. È un'operazione atomica se avviene sullo stesso disco.

**Sezione 6: Stream API**
*   Files.list(path) -> Restituisce uno Stream<Path> del contenuto di una directory (NON ricorsivo).
*   Files.walk(path) -> Restituisce uno Stream<Path> attraversando un albero di directory (ricorsivo).
*   stream.filter(lambda) -> Filtra lo stream tenendo solo gli elementi che soddisfano la condizione.
*   stream.forEach(lambda) -> Esegue un'azione per ogni elemento rimanente dello stream.

**Sezione 7: FileVisitor**
*   Files.walkFileTree(start, visitor) -> Attraversa un albero di file ed esegue le azioni definite nel visitor.
*   Metodi del Visitor (da sovrascrivere):
    *   preVisitDirectory(dir, attrs) -> Eseguito PRIMA di entrare in una directory.
    *   visitFile(file, attrs) -> Eseguito per OGNI file.
    *   postVisitDirectory(dir, exc) -> Eseguito DOPO aver visitato tutto il contenuto di una directory.
    *   visitFileFailed(file, exc) -> Eseguito se la visita a un file fallisce.

**Sezione 8: WatchService**
*   FileSystems.getDefault().newWatchService() -> Crea un nuovo servizio di monitoraggio.
*   path.register(service, ...events) -> Registra una directory per monitorare specifici eventi (CREATE, DELETE, MODIFY).
*   watchService.poll(timeout, unit) -> Attende per un tempo massimo un evento.
*   watchService.take() -> Attende all'infinito un evento (bloccante).
*   watchKey.pollEvents() -> Restituisce la lista degli eventi ricevuti.
*   watchKey.reset() -> FONDAMENTALE: Resetta la chiave per ricevere nuovi eventi.
*   watchService.close() -> Chiude il servizio di monitoraggio.
