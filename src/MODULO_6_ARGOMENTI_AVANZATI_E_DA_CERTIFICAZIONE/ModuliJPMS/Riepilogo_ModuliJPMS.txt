RIEPILOGO SUL JAVA PLATFORM MODULE SYSTEM (JPMS)

Il Java Platform Module System (JPMS), introdotto in Java 9, è un cambiamento fondamentale nel modo in cui le applicazioni Java vengono strutturate e distribuite. Il suo obiettivo è risolvere il problema del "classpath hell" e fornire un'incapsulamento più forte.

**Concetto Fondamentale:** Un **modulo** è un insieme di package correlati, progettato per essere riutilizzato. Ogni modulo dichiara esplicitamente:
1.  Le sue **dipendenze** (quali altri moduli gli servono per funzionare).
2.  I **package che espone** (quali parti del suo codice sono pubbliche per altri moduli).

--- STRUTTURA DI QUESTO ESEMPIO ---

Per mostrarti i concetti senza causare errori di compilazione in un progetto a modulo singolo, è stata adottata la seguente struttura:

1.  **Codice Eseguibile Semplificato:**
    - Le classi `MainApp` e `GreetingService` sono state messe nello **stesso package** (`com.main`) per permettere al progetto di compilare ed eseguire senza problemi.
    - Questo **NON** è come funzionerebbe un vero progetto modulare, ma serve a rendere l'esempio funzionante.

2.  **File di Esempio `.txt`:**
    - I file **`module-info.java.txt`** (nelle sottocartelle `app` e `service`) mostrano come sarebbero scritte le **vere dichiarazioni dei moduli**. Questi file sono la parte più importante da studiare per capire il JPMS.

--- IL FILE `module-info.java` (SPIEGAZIONE TEORICA) ---

Il cuore di un modulo è il file `module-info.java`.

**Direttive Principali:**

**1. `module nome.del.modulo { ... }`**
   Dichiara un modulo.

**2. `requires nome.altro.modulo;`**
   Dichiara una **dipendenza**. Il modulo corrente ha bisogno del codice esposto da `nome.altro.modulo`.

**3. `exports nome.del.package;`**
   Rende i tipi `public` del package `nome.del.package` accessibili ai moduli che dipendono da questo modulo. Per default, tutto ciò che è in un modulo è privato a quel modulo.

**Esempio di Struttura Multi-Modulo Reale (come descritto nei file `.txt`):**

`-- src`
`   |-- app.main`
`   |   |-- module-info.java`
`   |   `-- com/main/MainApp.java`
`   `-- app.service`
`       |-- module-info.java`
`       `-- com/service/GreetingService.java`

**`service/module-info.java`:**
`module app.service {`
`    exports com.service; // Rende pubblico il package com.service`
`}`

**`app/module-info.java`:**
`module app.main {`
`    requires app.service; // Dichiara che app.main dipende da app.service`
`}`

--- VANTAGGI DEL JPMS ---

1.  **Affidabilità della Configurazione:** L'applicazione non si avvia se mancano dei moduli.
2.  **Incapsulamento Forte:** Le classi interne di una libreria non sono accessibili per default.
3.  **Scalabilità e Performance:** Permette di creare immagini di runtime personalizzate (`jlink`) che includono solo i moduli strettamente necessari.
